Build an Access Control System (ACS)

XMind-ready project plan for a face-recognition-based access control platform with multi-branch support, role-based access, and modular project delivery.

Use each phase below as a main branch in XMind. The outline follows your plan.md and expands it with ACS-specific items visible in the app surface, so the diagram is easier to present and execute.
1
Planning & Requirements
Project kickoff, scope definition, team assignment, and delivery planning.
  • Kick-off meeting
  • Define user roles and permissions
  • Collect additional requirements
  • Task allocation meeting for breakdown and assignment
  • Frontend team: Member 1, Member 2
  • Backend team: Member 3, Member 4
  • Confirm tech stack
  • GitHub repository setup
  • Set project timeline and milestones
2
UI/UX Design & Database Design
Screen structure, visual planning, and data model preparation.
  • UI/UX design
    • Design the dark admin dashboard shell and responsive layout
    • Map the module flow: Module Hub, Dashboard, Users, Devices, Logs, Reports, Settings
    • Plan reusable table, card, dialog, badge, and filter patterns
    • Design auth screens, account management, and module license states
    • Design face capture preview, scan overlays, and result states
    • Design empty states, loading states, and no-results states
    • Define mobile and desktop behavior for each core screen
  • Database design
    • Design entities for accounts, users, employees, branches, devices, and doors
    • Design log tables for access events, device events, emergency events, and occupancy records
    • Define relationships between branch, device, door, and employee records
    • Define status fields and enums such as validity, device status, door type, and alarm severity
    • Prepare fields for face photo data, confidence score, reasons, and capture images
    • Define reporting fields for counts, timestamps, and daily summaries
  • Set up database environment and seed data
  • Define ERD and foreign key relationships
  • Prepare indexing and query strategy for logs and reports
3
Frontend Development
Build the ACS interface and connect all major screens in the app shell.
  • App shell and routing
  • Login and session management UI
  • Module Hub and access gating
    • Render ACS and VMS module cards
    • Show locked or licensed states per account
    • Support role-based account management for administrators
    • Support password change and sign out actions
  • Dashboard
    • Show live access summary cards
    • Display recent access feed
    • Show device and branch overview stats
    • Show face scan overlay in activity cards
  • User management
    • Register employees with photo upload
    • Track employee number, branch, and validity state
    • Search, filter, edit, and delete employees
    • Display last authentication time and registration time
  • Device management
    • Register face recognition terminals and door controllers
    • Track model, branch, location, IP address, firmware version, and daily scan count
    • Filter by office and device status
    • Edit and delete devices from the table view
  • Office / branch management
    • Manage office name, address, manager, phone, and email
    • Show employee count and device count per branch
    • Add, edit, search, and delete office records
  • Log management
    • Device log tab for connect, disconnect, restart, and firmware update events
    • Emergency log tab for intrusion, tamper, forced door, held door, and fire events
    • Authentication log tab for entry and exit records with confidence score
    • Occupancy log tab for daily hours and attendance tracking
    • Search, date filter, branch filter, and export actions
    • Face capture preview dialog for authentication records
  • Reports
    • Summary cards for access counts, granted, denied, and active users
    • Charts for daily access, office distribution, and peak hours
    • Top active users panel
    • CSV export action and period selector
  • Settings
    • General company profile settings
    • Notification preferences
    • Security settings such as 2FA, session timeout, and password policy
    • System backup and retention settings
4
Backend Development
Implement APIs, business rules, and database-backed ACS workflows.
  • Authentication and session APIs
    • Login and logout flow
    • Session persistence and current user lookup
    • Password change workflow
    • Account bootstrap from seeded access data
  • Role and module licensing APIs
    • Administrator, Operator, and Viewer access rules
    • ACS and VMS module licensing per account
    • Locked state when a module is not licensed
    • Admin-only account management actions
  • Employee and user APIs
    • Register employee name, number, branch, photo, and validity
    • Update employee status to valid, expired, or suspended
    • Store face enrollment photo data
    • Return last authentication and registration timestamps
  • Device and door APIs
    • Manage device name, model, branch, location, IP, and firmware version
    • Track online, offline, warning, and maintenance status
    • Link doors to devices and store door access levels
    • Track door lock and unlock state
  • Branch and office APIs
    • Store office contact information and manager details
    • Track employees and devices per office
    • Support active and inactive office status
  • Log and audit APIs
    • Authentication logs with entry, exit, granted, denied, confidence, and capture image
    • Device logs for connection, restart, and firmware update events
    • Emergency logs for intrusion, tamper, forced door, held door, and fire
    • Occupancy records with first entry, last exit, and total hours
    • Audit log support for configuration and system actions
  • Reports and export APIs
    • Summary metrics for access volume, granted, denied, and active users
    • Aggregated data for daily trends, office distribution, and peak hour analysis
    • CSV or spreadsheet export endpoints
  • Settings and configuration APIs
    • Notification settings
    • Security policy settings such as 2FA and session timeout
    • Backup and retention configuration
    • General company profile settings
  • Database services and validation
    • Record validation and relationship checks
    • Filtering and search support for list pages
    • Seed data and fixtures for offices, devices, users, and logs
5
Testing & QA
Validate each layer before release and close the project with bug fixing.
  • Frontend testing
  • Backend testing
  • Integration testing
  • Bug fixing and QA review
